The environment over the chaos

A project never loses information. It loses its state. The workspace can rebuild that state itself — on top of the tools the project already uses.

① Contributors

People · bots & agents · services. Work happens where it always did — nothing is filed twice, no one moves anywhere.

② Sources

Messenger, mail, calendar, tracker, knowledge base, repository. One append-only stream of observations — interpretations never rewrite the past.

③ Attach + Discovery, under the validation gate

Deterministic pipeline: hard identity anchors bind facts to topics, discovery finds new work; derived structure passes the environment's own rules. The LLM sees only the residual doubt.

④ Workspace — the environment

The unit — topics: a core that attracts, a halo that never does. One wrong attach stays one wrong attach. Death by default; rollback = one operation.
The property — trajectory: not just state but a movement vector. Activity is not progress.
The view — projections: one factual world; role views × attention tiers (focus · periphery · background).
The test is erasure: delete a topic, rebuild it from observations — nothing is lost.github.com/maaakso/environment-over-chaos
